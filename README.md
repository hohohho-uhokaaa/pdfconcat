# pdfconcat

2つの異なるディレクトリに存在する、同じファイル名（8桁の数字.pdf）のPDFファイルを自動でペアリングし、指定したモードで結合する.NETベースのコマンドラインツールです。

ドキュメントのスキャンデータ（奇数ページ・偶数ページが別々に出力されたものなど）を整理・結合する際などに便利です。

## ✨ 機能

- **自動ペアリング**: `dir1` と `dir2` の中から、「8桁の数字.pdf（例: `20260603.pdf`）」という規則に一致する同じ名前のファイルを自動で見つけて処理します。
- **2つの結合モード**:
  - `append`: 1つ目のPDFのバックスペースに、2つ目のPDFを丸ごと結合します。
  - `all`: 1つ目のPDFと2つ目のPDFのページを、1ページずつ**交互（インターリーブ）**に並べ替えて結合します。

## 🚀 動作環境

- .NET 10.0 SDK 
- 依存ライブラリ: `PdfSharpCore`

## 📦 準備
1. 本リポジトリをクローンまたはダウンロードします。
2. 依存パッケージを復元します。
   ```bash
   dotnet restore

🛠 使い方コマンドライン（ターミナル）から、以下の引数を指定して実行します。
Bash
dotnet run <dir1のパス> <dir2のパス> <モード(append|all)>

引数の説明引数説明
dir1のパス1つ目のPDFファイル群（all モード時は奇数ページ側を推奨）が格納されたディレクトリdir2のパス2つ目のPDFファイル群（all モード時は偶数ページ側を推奨）が格納されたディレクトリモード結合方法を append または all から指定（大文字小文字は区別しません）

💡 実行例
append モード（前後に丸ごと結合）
page1 フォルダー内のPDFの直後に、page2 フォルダー内の同じ名前のPDFをそのまま結合します。
Bash
dotnet run "/home/user/pdfconcat/page1" "/home/user/pdfconcat/page2" append

all モード（ページごとに交互結合）
page1 の1ページ目 ➔ page2 の1ページ目 ➔ page1 の2ページ目 ➔ page2 の2ページ目…… のようにシャッフルして結合します。スキャナーで表面・裏面を別々にスキャンしたデータの統合などに最適です。
Bash
dotnet run "/home/user/pdfconcat/page1" "/home/user/pdfconcat/page2" all

📂 出力先について
処理が成功すると、dir1（第1引数で指定したディレクトリ）の1つ上の階層に output というフォルダーが自動生成され、結合されたPDFが同じファイル名で保存されます。

Plaintextプロジェクトルート/
├── page1/       <- dir1 (12345678.pdf)
├── page2/       <- dir2 (12345678.pdf)
└── output/      <- ここに自動生成されます (12345678.pdf)
